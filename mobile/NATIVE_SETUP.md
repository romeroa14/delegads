# Native folders (Android / iOS)

These directories are generated by `flutter create .` and are intentionally
NOT committed — each developer runs that command locally with their
preferred platforms config.

## Bootstrapping

```bash
cd mobile
flutter create . \
  --project-name delegads_app \
  --org com.delegads \
  --platforms=ios,android
flutter pub get
```

This creates `android/`, `ios/`, `test/`, etc. without overwriting `lib/` or
`pubspec.yaml`.

## First-run checklist

1. `flutter doctor` — fix any reported issues
2. Android: open `android/` in Android Studio, accept SDK licenses
3. iOS: `cd ios && pod install && cd ..` (requires Xcode + CocoaPods)
4. Run on a device: `flutter run` (with a device or emulator attached)
5. Build release: `flutter build apk --release` or `flutter build ios --release`

## App IDs

- Android: `com.delegads.delegads_app`
- iOS bundle: `com.delegads.delegadsApp`

Adjust in `android/app/build.gradle` and `ios/Runner/Info.plist` if you
prefer different identifiers.
